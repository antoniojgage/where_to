An Overview of the Project

Application Concept

where to? allows the user to view the average salary of their occupation (or desired occupation) in major cities throughout the US. The user will also be able to view how these salaries compare to each city’s cost of living. This “bang for your buck” ratio is displayed on a heatmap, allowing the user to make an informed and strategic decision about where to move.


In addition to finding your next home base based on your occupation, where to? will allow users to compare the following data for two cities of their choice:


  • Consumer Price Index

  • Cost of Living + Rent Index

  • Restaurant Price Index

  • Rent Index

  • Groceries Index

  • Local Purchasing Power


Motivation Behind It

D3 is a popular data visualization library that is used by companies such as The New York Times. Plus, it looked like a lot of fun! We wanted to challenge ourselves to learn this powerful framework well enough to make something useful.

Design Process

  • where to? uses the MVC design structure.

  • It uses three different tables and left joins them to get all of the data needed.

  • Materialize was used for the grid and sleek components.

  • Photoshop and Illustrator were used to create our logo and edit photos of the cities to use in the center of the donut charts

  • The images courtesy of unsplash.com.

Technologies Used

D3

"D3 allows you to bind arbitrary data to a Document Object Model (DOM), and then apply data-driven transformations to the document."

MySQL/ Amazon Web Services

2015 salary data pulled from U.S. Bureau of Labor Statistics: https://www.bls.gov/developers/


2015 CPI data pulled from Numbeo: https://www.numbeo.com/cost-of-living/rankings.jsp

Sequelize and Sequelize-Auto

We used sequelize orm, but our tables were created before the application. Sequelize-Auto made it easy to generate our models based on our already existing tables.
